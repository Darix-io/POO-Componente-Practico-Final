# Informe Sprint 3 - Modelado UML

## Modelado UML

Durante este sprint se documentó el diseño inicial del sistema mediante diagramas UML orientados a la arquitectura propuesta del proyecto.

### 1. Diagrama de clases - Security
Se modeló el módulo de autenticación y usuarios con el modelo User, el manager y la relación con permisos y grupos.

### 2. Diagrama de clases - Módulo QR
Se definió la relación entre usuarios, códigos QR y registros de acceso.

### 3. Diagramas de secuencia
- Login completo
- Validación QR
- Generación y renovación QR

### 4. DER completo
Se establecieron las entidades principales, sus claves primarias, claves foráneas y restricciones de unicidad.

### 5. Diagrama de despliegue
Se representó la arquitectura de ejecución del sistema: navegador, frontend, backend y base de datos.

### 6. Diagrama de casos de uso
Se describieron las acciones principales del administrador y del usuario final.

## Integración recomendada
Los diagramas pueden exportarse a imágenes PNG/SVG o integrarse directamente en el informe final usando Markdown con Mermaid.
