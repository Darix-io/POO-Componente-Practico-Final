# Roadmap del proyecto Modelo QR UNEMI

## Sprint 1 - Base del proyecto
- Crear repositorio con estructura backend/, frontend/ y docs/
- Configurar entorno virtual Python 3.12 y proyecto base
- Configurar settings.py con python-decouple y .env
- Verificar runserver y commit inicial

## Sprint 2 - Modelado inicial
- Crear app principal del sistema
- Definir modelos base para usuarios, accesos y QR
- Preparar migraciones iniciales

## Sprint 3 - Autenticación y roles
- Implementar login y registro básico
- Definir roles de administrador y usuario
- Configurar permisos por rol

## Sprint 4 - Generación y escaneo de QR
- Crear módulo para generar códigos QR
- Implementar lectura de QR desde la app
- Vincular QR con registros de acceso

## Sprint 5 - Control de acceso
- Registrar entradas y salidas
- Validar horarios y permisos
- Registrar eventos de acceso

## Sprint 6 - Panel administrativo
- Crear dashboard para administración
- Visualizar usuarios, accesos y reportes
- Gestión básica de registros

## Sprint 7 - Reportes y estadísticas
- Generar reportes por fecha y usuario
- Mostrar métricas de asistencia
- Exportar información básica

## Sprint 8 - Frontend mejorado
- Mejorar interfaz de usuario
- Integrar vistas de login, dashboard y reportes
- Optimizar experiencia de uso

## Sprint 9 - Seguridad y validaciones
- Fortalecer autenticación
- Validar entradas de datos
- Mejorar manejo de errores

## Sprint 10 - Pruebas y calidad
- Añadir pruebas unitarias e integradas
- Corregir bugs detectados
- Validar flujo completo del sistema

## Sprint 11 - Despliegue y entrega
- Preparar entorno de producción
- Configurar variables de entorno finales
- Entregar proyecto documentado y funcional
